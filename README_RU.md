# Инструмент резервного копирования Amnezia

[English](README.md) | [Русский](README_RU.md)

Создаёт резервные копии и восстанавливает данные контейнеров Amnezia VPN.

## Что делает скрипт

- Создаёт резервные копии пользователей с отметкой даты и времени
- Позволяет восстановить данные из созданных бэкапов при необходимости

## Как использовать

### Создание резервной копии
```bash
bash ./amnezia-backup.sh [каталог_резервных_копий]
```
Если **каталог_резервных_копий** не указан, используется текущая рабочая директория.

### Восстановление из резервной копии
```bash
bash ./amnezia-backup.sh -r [каталог_резервных_копий]
```

## Настройка

Вы можете изменить значения по умолчанию с помощью переменных окружения:

- `CONTAINER_PREFIX`: Префикс контейнеров для резервного копирования (по умолчанию: `amnezia`)
- `RETENTION_COUNT`: Количество старых бэкапов для хранения (по умолчанию: `5`)
- `ROLLBACK_RETENTION_COUNT`: Количество «страховочных» бэкапов перед восстановлением (по умолчанию: `2`)
- `CONSISTENT_BACKUP`: Если `true`, контейнер будет приостановлен во время бэкапа (по умолчанию: `false`)

Пример:
```bash
CONSISTENT_BACKUP=true bash ./amnezia-backup.sh /path/to/backups
```

## Требования

- Установленный и запущенный Docker
- Этот файл скрипта
- Достаточно свободного места на диске для хранения бэкапов

## Возможности

- **Атомарные бэкапы:** Резервные копии сначала создаются во временных файлах и перемещаются только после успешного завершения.
- **Безопасные права доступа:** Файлы бэкапов создаются с ограниченными правами доступа (600).
- **Очистка:** Автоматическое удаление временных файлов при завершении или прерывании скрипта.
- **Хранение версий:** Автоматически сохраняются только последние бэкапы и страховочные снимки.
- **Проверка зависимостей и ресурсов:** Проверяется наличие `docker`, `tar` и достаточного места на диске.
- **Консистентные бэкапы:** Возможность приостановить контейнер во время создания резервной копии для лучшей целостности данных.
- **Безопасное восстановление:** Перед восстановлением автоматически создаётся страховочный «pre-restore» бэкап. В случае ошибки ваши данные останутся сохранены.

## Важно

- Восстановление перезапустит ваши контейнеры
- Тестируйте восстановление заранее
- Храните резервные копии в безопасном месте
